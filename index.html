Security first

Recommended production shape

  • Keep the frontend static and the API/database private.
  • Refresh the database on a schedule instead of from public requests.
  • Restrict cross-origin access to the exact frontend origin you deploy.
  • Use separate database credentials for the read-only API and the ingestion job.
  • Place the API behind TLS, rate limits, and a WAF on your hosting platform.

Hosting options

Where to run this

  • Best balance: Cloudflare Pages or Vercel for the frontend, Fly.io or Render for the Plumber API, and Neon/Supabase/PostgreSQL for the database.
  • Cheapest first deploy: one small Fly.io or Render service with the Plumber API and a persisted SQLite file.
  • Keep the GitHub Pages domain: host only the frontend here and point it to a separate API origin with a strict CORS allow-list.
