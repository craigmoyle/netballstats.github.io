# Architecture Overview — netballstats

## System Description

`netballstats` is a publicly accessible Super Netball historical-statistics archive.  A static
frontend (Azure Static Web Apps / SWA) proxies all API calls through its `/api/*` route to a
read-only R Plumber REST service running on Azure Container Apps (ACA).  Player and team data
are stored in Azure Database for PostgreSQL Flexible Server and refreshed weekly by scheduled
ACA jobs that pull from the Champion Data API via the `superNetballR` R package.  Browser
usage telemetry is collected by the Plumber API acting as a privacy-safe proxy before forwarding
sanitised events to Azure Application Insights.

## Security Posture Inventory

| Control | Implementation | Source |
|---|---|---|
| Rate limiting | Per-IP, per-process, 60 req/60 s; left-most X-Forwarded-For; prune every 10 min | `api/plumber.R` ~L58 |
| Parameterised SQL | `DBI::sqlInterpolate` throughout all query builders | `api/R/helpers.R` |
| Stat allowlist validation | `validate_stat()` + `safe_stat_in_sql()` (`^[A-Za-z0-9]+$`) | `api/R/helpers.R` ~L282, L1745 |
| CORS | Restricted to SWA hostname via `NETBALL_STATS_ALLOWED_ORIGINS` | `api/R/helpers.R` ~L494 |
| Security response headers | `X-Content-Type-Options`, `X-Frame-Options: DENY`, `Referrer-Policy`, `Permissions-Policy`, `Cache-Control: no-store` | `api/plumber.R` ~L487 |
| Content Security Policy | `staticwebapp.config.json`; `script-src` from `self` + App Insights CDN; no SRI | `staticwebapp.config.json` |
| Non-root container | uid=1000 `netballstats` user | `Dockerfile.azure` |
| TLS enforced | `sslmode=require` on all PostgreSQL connections | `R/database.R` ~L65,79 |
| Read-only API user | `netballstats_api` has SELECT-only grants; no write endpoints | `infra/modules/app-stack.bicep` |
| Managed identity | Key Vault and ACR access via `userAssignedIdentity` — no credentials in environment | `infra/modules/app-stack.bicep` |
| Key Vault secrets | DB passwords stored as Key Vault secrets; referenced as secret refs in ACA config | `infra/modules/app-stack.bicep` |
| ACR hardening | `adminUserEnabled=false`, `anonymousPullEnabled=false` | `infra/modules/app-stack.bicep` |
| OIDC federated credentials | GitHub Actions ACR cleanup uses OIDC — no long-lived secrets | `.github/workflows/cleanup-registry.yml` |
| Telemetry privacy | fetch/XHR auto-tracking disabled; raw query text not logged; page type + event name allowlisted | `api/plumber.R`, `assets/telemetry.js` |

## Sequence Diagrams

### SQ-01 — User Page Load and Stats Query

```
EndUser ->> SWAFrontend : HTTPS GET / (page + static assets)
SWAFrontend -->> EndUser : HTML/JS bundle
EndUser ->> SWAFrontend : HTTPS GET /api/meta
SWAFrontend ->> PlumberAPI : Proxy /meta
PlumberAPI -->> SWAFrontend : JSON { schema_version, browser_telemetry: { connection_string } }
SWAFrontend -->> EndUser : meta JSON (connection_string exposed to browser)
EndUser ->> SWAFrontend : HTTPS GET /api/players?stat=goalAssists&season=2024
SWAFrontend ->> PlumberAPI : Proxy /players
PlumberAPI ->> PostgreSQL : TLS/SQL parameterised SELECT
PostgreSQL -->> PlumberAPI : result rows
PlumberAPI -->> SWAFrontend : JSON player stats array
SWAFrontend -->> EndUser : rendered table
```

### SQ-02 — Browser Telemetry Proxy

```
EndUser ->> SWAFrontend : HTTPS POST /api/telemetry (pageView / customEvent)
SWAFrontend ->> PlumberAPI : Proxy POST /telemetry
PlumberAPI ->> PlumberAPI : validate event_type (allowlist), sanitise page_name
PlumberAPI ->> AppInsights : HTTPS POST telemetry envelope (includes client_ip)
AppInsights -->> PlumberAPI : 200 OK
PlumberAPI -->> SWAFrontend : 200 OK
```

### SQ-03 — Weekly Database Refresh

```
AzureAD -->> DBRefreshJob : Managed identity token (IMDS)
DBRefreshJob ->> KeyVault : HTTPS GET admin password secret
KeyVault -->> DBRefreshJob : admin password
DBRefreshJob ->> ChampionData : HTTPS GET season stats (superNetballR)
ChampionData -->> DBRefreshJob : raw stats JSON
DBRefreshJob ->> PostgreSQL : TLS/SQL DROP / CREATE / INSERT (admin credentials)
PostgreSQL -->> DBRefreshJob : success
```

## Deployment Context

- **Classification**: NETWORK_SERVICE
- **Platform**: Azure Static Web Apps (frontend) + Azure Container Apps (API, jobs)
- **Public exposure**: SWAFrontend and PlumberAPI are externally reachable on 443 with no authentication; Tier-1 findings are in scope.
- **Image supply chain**: GitHub Actions pushes to Azure Container Registry; ACA pulls on deploy; no image signing configured.
- **Private networking**: PostgreSQL private-networking mode is disabled by default in the Bicep parameterisation; public endpoint is enabled with an "Allow Azure Services" firewall rule.
